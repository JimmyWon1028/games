# AGENTS.md

## 回覆與程式風格

- 一律用繁體中文回覆使用者。
- 代碼注釋使用英文。
- 程式碼縮排一律使用 2 個空白，不使用 tab。
- 修改既有檔案時，先跟隨該檔案原本的命名、排版與互動風格；除非任務要求，避免大規模重排。

## 專案概覽

- 這是一個純靜態的 HTML 遊戲合集，入口是 `index.html`。
- 每個遊戲目前都是 `games/` 底下獨立的 `.html` 檔，CSS 與 JavaScript 多數直接內嵌在同一檔案中。
- 圖片資產集中放在 `assets/images/`，音效資產集中放在 `assets/audio/`。
- 專案沒有 npm、打包器或框架依賴；維持「開啟即玩」與無外部依賴的特性。
- README 使用繁體中文描述專案；首頁與部分舊遊戲文字仍混有簡體中文，若修改使用者可見文案，優先轉為繁體中文並保持遊戲語氣一致。

## 開發注意事項

- 新增遊戲時，同步更新：
  - `index.html` 的遊戲卡片與搜尋關鍵字。
  - `README.md` 的遊戲列表與專案結構。
- 新增或移動圖片時，放在 `assets/images/` 並同步修正 README 或 HTML 的相對路徑。
- 新增或移動音效時，放在 `assets/audio/`；檔名若有空白，HTML 內使用 URL encoded 路徑。
- 修改遊戲邏輯時，注意鍵盤、滑鼠、觸控操作是否仍可用。
- 使用 Canvas 的遊戲通常以 `requestAnimationFrame` 驅動主循環；避免引入額外計時器造成重複循環。
- 使用 Web Audio API 的遊戲要保留使用者互動後才啟動音訊的瀏覽器相容性。
- 使用 `localStorage` 儲存分數時，避免改壞既有 key，除非有明確遷移理由。
- 維持單檔遊戲的可攜性，不新增外部圖片、音效或套件，除非使用者明確要求。

## 視覺與互動

- 遊戲大廳維持暗色街機／太空風格，卡片、搜尋與隨機遊玩功能要保持可用。
- 遊戲畫面需在桌面與手機寬度下不溢出；固定尺寸 Canvas 應搭配 responsive CSS。
- 按鈕、HUD、分數與提示文字不能遮擋主要遊戲區。

## 驗證方式

- 這個專案可直接用靜態檔伺服器測試：

```bash
python3 -m http.server 8080
```

- 開啟 `http://localhost:8080` 檢查遊戲大廳。
- 修改單一遊戲後，至少手動確認：
  - 頁面能載入，瀏覽器 console 無明顯錯誤。
  - 開始、暫停、重新開始或主要操作流程可用。
  - 桌面鍵盤操作與手機觸控操作沒有被破壞。

## Git 注意事項

- 工作樹可能已有使用者修改；不要還原自己沒有做的變更。
- 不要主動 push 到遠端，除非使用者明確要求。
